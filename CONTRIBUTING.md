# Contributing to Gem Clip

Thank you for your interest in contributing to Gem Clip! We welcome all contributions, from bug reports to new features. To ensure a smooth process, please review these guidelines.

## How to Contribute

- **Reporting Bugs:** If you find a bug, please open an issue and provide detailed information, including steps to reproduce, expected behavior, and actual behavior.
- **Suggesting Enhancements:** For new features or improvements, open an issue to discuss your ideas before you start working on them. This helps ensure your contribution aligns with the project's goals.
- **Submitting Pull Requests:** If you want to contribute code, please follow the steps below.

## Pull Request Process

1.  **Fork the repository** and create your branch from `main`.
2.  **Make your changes.** Please adhere to the coding style and conventions used in the project (see `README.md` for details).
3.  **Add tests** for your changes if applicable. Run all tests to ensure nothing is broken.
    ```bash
    python -m pytest -q
    ```
4.  **Update the documentation** (`README.md`, etc.) if your changes affect it.
5.  **Ensure your commit messages are clear** and follow the Conventional Commits specification (e.g., `feat:`, `fix:`, `docs:`).
6.  **Submit a pull request** with a clear description of the changes and why they are needed. Link to any relevant issues.

## Development Setup

For details on how to set up your development environment, please see the "Installation" and "Development" sections in the [README.md](README.md) file.

We look forward to your contributions!
